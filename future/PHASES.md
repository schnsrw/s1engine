## Phase 2: Embeddable Editor SDK

**Goal**: Transform the editor from a standalone app into an embeddable component with a clean JavaScript API.

### Deliverables

#### 2.1 Core SDK (`@rudra/sdk`)
- [ ] **S1Engine class** (wraps WasmEngine)
  ```typescript
  class S1Engine {
    static async init(): Promise<S1Engine>
    create(): S1Document
    open(data: ArrayBuffer): S1Document
    openUrl(url: string): Promise<S1Document>
    detectFormat(data: ArrayBuffer): Format
  }
  ```

- [ ] **S1Document class** (wraps WasmDocument)
  ```typescript
  class S1Document {
    // Content
    toHTML(): string
    toPaginatedHTML(config?: LayoutConfig): string
    toPlainText(): string

    // Export
    export(format: Format): ArrayBuffer
    exportBlob(format: Format): Blob
    exportDataUrl(format: Format): string

    // Editing
    insertText(nodeId: string, offset: number, text: string): void
    deleteText(nodeId: string, offset: number, length: number): void
    formatSelection(range: SelectionRange, key: string, value: any): void
    insertTable(afterNodeId: string, rows: number, cols: number): void
    insertImage(afterNodeId: string, data: ArrayBuffer, mime: string): void

    // Metadata
    get title(): string
    set title(value: string)
    get pageCount(): number
    get wordCount(): number

    // Events
    on(event: 'change' | 'save' | 'export', callback: Function): void
    off(event: string, callback: Function): void

    // Collaboration
    enableCollab(config: CollabConfig): CollabSession
  }
  ```

- [ ] **TypeScript definitions** — Full type coverage
- [ ] **Event system** — EventEmitter with typed events
- [ ] **Error handling** — Typed errors with codes and messages

#### 2.2 Editor Component (`@rudra/editor`)
- [ ] **S1Editor class** — Main entry point
  ```typescript
  class S1Editor {
    static async create(
      container: HTMLElement,
      options?: EditorOptions
    ): Promise<S1Editor>

    // Document
    open(data: ArrayBuffer): void
    openUrl(url: string): Promise<void>
    createNew(): void
    getDocument(): S1Document

    // Editor state
    get isDirty(): boolean
    get isReadOnly(): boolean
    set readOnly(value: boolean)

    // Toolbar
    setToolbar(items: ToolbarItem[]): void
    addToolbarItem(item: ToolbarItem, position?: number): void
    removeToolbarItem(id: string): void

    // Theme
    setTheme(theme: Theme | string): void

    // Events
    on(event: EditorEvent, callback: Function): void

    // Lifecycle
    destroy(): void
  }
  ```

- [ ] **EditorOptions** configuration
  ```typescript
  interface EditorOptions {
    // Appearance
    theme?: 'default' | 'dark' | 'minimal' | Theme
    locale?: string
    toolbar?: ToolbarConfig | false     // false = hidden
    statusBar?: boolean
    ruler?: boolean
    pageView?: boolean                  // Paginated vs continuous

    // Behavior
    readOnly?: boolean
    autosave?: AutosaveConfig | false
    spellcheck?: boolean

    // Collaboration
    collab?: {
      serverUrl: string
      roomId: string
      userName: string
      userColor?: string
      token?: string                    // JWT for auth
    } | false

    // File handling
    acceptFormats?: Format[]            // Which formats can be opened
    exportFormats?: Format[]            // Which formats can be exported
    maxFileSize?: number                // bytes

    // Callbacks
    onReady?: () => void
    onChange?: (event: ChangeEvent) => void
    onSave?: (doc: S1Document) => void | Promise<void>
    onError?: (error: S1Error) => void

    // White-label (see WHITE_LABELING.md)
    branding?: BrandingConfig
  }
  ```

- [ ] **Toolbar configuration**
  ```typescript
  // Preset toolbars
  S1Editor.toolbars.full        // All buttons (default)
  S1Editor.toolbars.standard    // Common buttons
  S1Editor.toolbars.minimal     // Bold, italic, headings only

    items: [
      'undo', 'redo', '|',
      'bold', 'italic', 'underline', '|',
      'heading', 'font-family', 'font-size', '|',
      'align-left', 'align-center', 'align-right', '|',
      'bullet-list', 'numbered-list', '|',
      'insert-table', 'insert-image', '|',
      'export-pdf', 'export-docx',
      '|',
      { id: 'custom-save', label: 'Save', icon: 'save-icon', onClick: handleSave }
    ]
  }
  ```

## Phase 3: Server API (REST + Storage)

**Goal**: Build a Rust-based HTTP server providing REST API for document management, format conversion, and storage.

### Deliverables

#### 3.1 Server Crate Setup
- [ ] Create `crates/s1-server/` (or `server/`)
- [ ] Set up Axum with Tokio runtime
- [ ] Configuration system (TOML + env vars)
- [ ] Structured logging (tracing)
- [ ] Health check endpoint (`GET /health`)
- [ ] OpenAPI spec generation (utoipa)
- [ ] CORS middleware
- [ ] Request ID middleware (for tracing)

#### 3.2 Document CRUD API
```
POST   /api/v1/documents                    Create document (upload or empty)
GET    /api/v1/documents                    List documents (paginated, filtered)
GET    /api/v1/documents/:id               Get document metadata
GET    /api/v1/documents/:id/content        Download document bytes
PUT    /api/v1/documents/:id               Update document (re-upload)
PATCH  /api/v1/documents/:id/metadata      Update metadata only
DELETE /api/v1/documents/:id               Delete document
```

- [ ] Implement all endpoints
- [ ] Request validation (file size, format, required fields)
- [ ] Pagination (cursor-based for list)
- [ ] Filtering (by format, date, title search)
- [ ] Sorting (by created, updated, title)
- [ ] ETag / If-Match for optimistic concurrency
- [ ] Content negotiation (JSON metadata vs raw bytes)

#### 3.3 Format Conversion API
```
POST   /api/v1/convert                      Stateless conversion
POST   /api/v1/documents/:id/export/:format Export stored document to format
GET    /api/v1/documents/:id/preview        Get HTML preview
GET    /api/v1/documents/:id/thumbnail      Get page thumbnail (PNG)
GET    /api/v1/documents/:id/text           Get plain text extraction
```

- [ ] Stateless conversion (upload bytes, get converted bytes)
- [ ] Supported conversions:
  - DOCX → PDF, ODT, TXT, MD, HTML
  - ODT → PDF, DOCX, TXT, MD, HTML
  - TXT → DOCX, ODT, PDF
  - MD → DOCX, ODT, PDF, HTML
- [ ] Conversion options (page size, margins, fonts)
- [ ] Async conversion for large files (return job ID, poll for result)
- [ ] Thumbnail generation (render first page to PNG)

#### 3.4 Storage Backend
- [ ] `StorageBackend` trait definition
- [ ] **LocalStorage** implementation
  - Documents stored as files: `/data/{tenant_id}/{doc_id}.{format}`
  - Metadata stored as JSON sidecar: `/data/{tenant_id}/{doc_id}.meta.json`
- [ ] **S3Storage** implementation
  - Compatible with AWS S3, MinIO, Cloudflare R2, DigitalOcean Spaces
  - Configurable bucket, prefix, region
  - Presigned URLs for direct client upload/download
- [ ] **PostgresStorage** implementation
  - Schema: `documents` table with bytea content
  - Full-text search on metadata
  - Migration system (sqlx migrate)
- [ ] **MemoryStorage** implementation (for testing)

#### 3.5 Webhook System
```
POST   /api/v1/webhooks                     Register webhook
GET    /api/v1/webhooks                     List webhooks
DELETE /api/v1/webhooks/:id                 Delete webhook
```

- [ ] Webhook events:
  - `document.created` — new document uploaded/created
  - `document.updated` — document content changed
  - `document.deleted` — document deleted
  - `document.exported` — document exported to another format
  - `collab.session_started` — collaboration room opened
  - `collab.session_ended` — all peers left room
- [ ] Webhook delivery:
  - HTTP POST with JSON payload
  - HMAC signature verification (X-S1-Signature header)
  - Retry with exponential backoff (3 retries)
  - Delivery logs

#### 3.6 Docker Image
- [ ] Multi-stage Dockerfile for `s1-server`
- [ ] Docker Compose with server + storage (Postgres + MinIO)
- [ ] Environment variable documentation

### Exit Criteria
- `docker run s1engine/server` starts API server
- All CRUD endpoints work with local storage
- Format conversion works for all supported paths
- OpenAPI spec auto-generated and accessible at `/api/docs`
- Integration tests cover all endpoints

## Phase 5: Auth, Permissions & Multi-tenancy

**Goal**: Add authentication, document-level permissions, and tenant isolation.

### Deliverables

#### 5.1 Authentication Middleware
- [ ] **JWT validation**
  - HS256 and RS256 support
  - Configurable secret/public key
  - Token expiry validation
  - Custom claims extraction
  ```json
  {
    "sub": "user-123",
    "name": "Alice",
    "tenant_id": "org-456",
    "permissions": {
      "doc-789": "write",
      "doc-012": "read"
    },
    "exp": 1700000000
  }
  ```

- [ ] **API key validation**
  - Header: `X-API-Key: sk_live_...`
  - Key lookup (in-memory cache, backed by storage)
  - Per-key rate limits
  - Key rotation support

- [ ] **Anonymous access**
  - Configurable: allow/deny
  - Read-only by default for anonymous
  - Rate limited

#### 5.2 Permission System
- [ ] **Document permissions**
  - `owner` — full control (delete, share, transfer)
  - `admin` — edit, export, manage permissions
  - `editor` — edit, export
  - `commenter` — read, add comments
  - `viewer` — read-only
- [ ] **Permission enforcement**
  - Checked on every API call
  - Checked on WebSocket operations
  - Inherited from tenant defaults
- [ ] **Sharing API**
  ```
  POST   /api/v1/documents/:id/share      Add permission
  DELETE /api/v1/documents/:id/share/:uid  Remove permission
  GET    /api/v1/documents/:id/share       List permissions
  ```

#### 5.3 Multi-tenancy
- [ ] **Tenant isolation**
  - Every document belongs to a tenant
  - Queries are always tenant-scoped
  - Storage is tenant-prefixed (S3: `{bucket}/{tenant_id}/`, Postgres: WHERE clause)
- [ ] **Tenant configuration**
  - Per-tenant storage quota
  - Per-tenant rate limits
  - Per-tenant feature flags
  - Per-tenant white-label config (see Phase 6)
- [ ] **Tenant extracted from JWT** (no separate tenant management API — consumer manages tenants)

#### 5.4 Audit Logging
- [ ] Log all document access (who, when, what, from where)
- [ ] Log all permission changes
- [ ] Log all exports
- [ ] Structured JSON logs (machine-parseable)
- [ ] Optional: ship to consumer's logging system via webhook

### Exit Criteria
- JWT authentication works on all endpoints
- API key authentication works
- Document permissions enforced (CRUD + collab)
- Multi-tenant isolation verified (cross-tenant access denied)
- Audit logs emitted for all operations

## Risk Register

| Risk | Impact | Likelihood | Mitigation |
|------|--------|------------|------------|
| WASM bundle too large | Slow initial load | Medium | Tree-shaking, lazy loading, CDN caching |
| CRDT conflicts at scale | Data loss | Low | Extensive convergence tests (already 182+) |
| Format fidelity gaps | User complaints | Medium | Focus on 90% case, clear documentation of limitations |
| Contributor adoption slow | Project stalls | Medium | Good docs, easy contribution path, responsive issues |
