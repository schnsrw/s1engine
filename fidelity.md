Format Fidelity + Markdown Extension                                                                                                                                                                                       │
│                                                                                                                                                                                                                            │
│ Context                                                                                                                                                                                                                    │
│                                                                                                                                                                                                                            │
│ The repo has strong DOCX fidelity (formatting, tables, images, lists, sections, headers/footers, hyperlinks, bookmarks, comments, tab stops, paragraph borders/shading, character spacing, superscript/subscript). ODT     │
│ currently covers ~50% of those features. TXT strips all structure. Markdown does not exist yet. The user wants to close fidelity gaps across ODT and TXT, and add Markdown as a new format.                                │
│                                                                                                                                                                                                                            │
│ The model layer already has all necessary NodeTypes (BookmarkStart, BookmarkEnd, CommentStart, CommentEnd, CommentBody, Header, Footer, Section) and AttributeKeys (Superscript, Subscript, FontSpacing, ParagraphBorders, │
│ Background, TabStops, HyperlinkUrl, HyperlinkTooltip, BookmarkName, CommentId, CommentAuthor, CommentDate). No model changes needed.                                                                                       │
│                                                                                                                                                                                                                            │
│ Implementation Order                                                                                                                                                                                                       │
│                                                                                                                                                                                                                            │
│ 1. F.1: ODT Quick Wins — property-level additions (no new XML elements)                                                                                                                                                    │
│ 2. F.2: Markdown Format Crate — new s1-format-md with pulldown-cmark                                                                                                                                                       │
│ 3. F.3: ODT Hyperlinks + Bookmarks — content-level element additions                                                                                                                                                       │
│ 4. F.4: ODT Tab Stops + Paragraph Borders — property + content mix                                                                                                                                                         │
│ 5. F.5: TXT Fidelity — heading/list markers in plain text output                                                                                                                                                           │
│ 6. F.6: ODT Comments — new parser/writer modules                                                                                                                                                                           │
│ 7. F.7: ODT Headers/Footers/Sections — structural additions                                                                                                                                                                │
